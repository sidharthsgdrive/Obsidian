The time complexity of the given function is **O(n)**, where `n` is the input value. Here's a step-by-step explanation:

### Code Analysis:
```c
void change(int n) {
    int i = 0, j = 0;
    while (i < n) {      // Outer loop
        n = n / 2;       // Halve `n` (integer division)
        j = 0;
        while (j < n) {  // Inner loop
            j++;
        }
    }
}
```

### Key Observations:
1. **Outer Loop Behavior**:
   - The outer loop runs as long as `i < n`. Since `i` remains `0` and `n` is halved in each iteration, the outer loop executes approximately **log₂(n)** times (specifically, until `n` becomes 0).
   - Example: For `n = 8`, the sequence of `n` is `8 → 4 → 2 → 1 → 0` (4 iterations).

2. **Inner Loop Work**:
   - In each outer iteration, after halving `n`, the inner loop runs exactly `n` times (where `n` is the current halved value).
   - The work done by the inner loop across all outer iterations forms the series:   $\left\lfloor \frac{n}{2} \right\rfloor + \left\lfloor \frac{n}{4} \right\rfloor + \left\lfloor \frac{n}{8} \right\rfloor + \cdots + 0$


### Summing the Inner Loop Iterations:
The series above is bounded by:
$\frac{n}{2} + \frac{n}{4} + \frac{n}{8} + \cdots \leq n$

This is a geometric series converging to **n**. Specifically:
- The sum is exactly **n - b(n)**, where **b(n)** is the number of 1s in the binary representation of `n` (which is at most **O(log n)**).
- Thus, the total work is **Θ(n)**.

### Example:
- For **n = 8**:
  - Outer Iteration 1: `n = 8/2 = 4` → Inner loop runs 4 times.
  - Outer Iteration 2: `n = 4/2 = 2` → Inner loop runs 2 times.
  - Outer Iteration 3: `n = 2/2 = 1` → Inner loop runs 1 time.
  - Outer Iteration 4: `n = 1/2 = 0` → Inner loop skipped.
  - **Total inner iterations: 4 + 2 + 1 = 7 ≈ n**.

### Why O(n)?
- The outer loop runs **O(log n)** times, but the total work is dominated by the inner loops, which sum to **n**.
- The series sum **n/2 + n/4 + ...** is asymptotically **O(n)** since it converges to **n**.

### Conclusion:
The function has a **linear time complexity O(n)** due to the cumulative work of the inner loops across all outer iterations. The halving of `n` reduces the problem size geometrically, but the total operations remain proportional to `n`.
